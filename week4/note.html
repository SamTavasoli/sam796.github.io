Summary of my study in Week 4

  • Object-oriented programming is often used to model representations of objects in the real world. is a way of programming that uses objects that encapsulate their own properties and methods
  • I learned more about three concept:
    1. Encapsulation
    2. Polymorphism
    3. Inheritance
  • Constructor functions can be used to create instances of objects.
  • ES6 introduced class declarations that use theclasskeyword. These can be used in place of constructor functions.
  • An alternative way to create objects is to use aconstructor function
  • The keyword "this" is used to represent the object that will be returned by the constructor function
  • All instances of a class or constructor function inherit all the properties and methods of its prototype.
  • The names of constructor functions or class declarations are capitalized
  • All objects have aconstructorproperty that returns the constructor function that created it
  • The static keyword can be used in class declarations to create a static method
  • Every class has a prototype property that is shared by every instance of the class. So any properties or methods of a classâ€™s prototype can be accessed by every object instantiated by that class
  • All classes and constructor functions have a prototype property that returns an object
  • The prototype can be used to add any new properties and methods after the class has been declared
  • Public & Private methods
  • Private properties and methods can be created by defining variables using const and defining a function inside a constructor function. These can be made public using getter and setter functions.
  • Monkey-patching is the process of adding methods to built-in objects by augmenting their prototypes. This should be done with caution as it can cause unexpected behavior in the way built-in objects work.
  • A class can inherit from another class using the "extends" keyword in a class declaration.
  • The concept of polymorphism means that different objects can have the same method, but implement it in different ways
  • Polymorphism allows objects to override shared methods with a more specific implementation.
  • Methods can be borrowed from other objects.
