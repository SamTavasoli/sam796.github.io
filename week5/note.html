Errors

When we encounter with error, it can be because of three reasons:

  1. System error: problem with the system or external devices
  2. Programmer error: incorrect syntax or faulty logic
  3. User error: the user has entered data incorrectly

Exceptions

An exception is an error that produces a return value that can then be used by the program to deal with the error.
An exception will produce stack trace too.
A stack trace will work backwards from the point at which the error occurred to identify the original function or method that started the sequence.
In stack trace, we work backwards to see that the error has been caused by what in the first place.

Warnings

A warning can occur if there’s an error in the code that isn't enough to cause the program to crash. Warnings and exceptions are presented differently in various environments.
Some browsers will show a small icon in the corner of the browser window to indicate that an exception or warning has occurred.
Others require the console to be open to see any warnings or exceptions.

Testing

Writing good tests means your code will be less brittle as it develops, and any errors will be identified early on.

  • Test-driven development(TDD) is the process of writing tests before any actual code.
  • Testing frameworks provide a structure to write meaningful tests and then run them.

What is Jest?
Jestis a TDD framework, created by Facebook, that has gained a lot of popularity recently. It makes it easy to createS
