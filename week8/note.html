CSS3 Transforms and Transitions

- The CSS3 transform property lets you lets us translate, rotate, scale, and/or skew any element on the page.
- Translation functions allow you to move elements left, right, up, or down. These functions are similar to the
behavior of position: relative; when declaring top and left, moving elements up and down or left and right
along the x and y axes.
- The translate(x,y) function moves an element x from the left, and y from the top.
- The scale(x,y) function scales an element by the defined factors horizontally then vertically. If only one
value is provided, it will be used for both the x and y values, growing or shrinking your element or pseudo-element
while maintaining the original aspect ratio.
- The rotate() function rotates an element around the point of origin by a specified angle value.
- The skew(x,y) function specifies a skew along the x and y axes.
- The steps to create a simple Transition using only CSS:

  1. Declare the original state of the element in the default style declaration.
  2. Declare the final state of your transitioned element; for example, a :hover state.
  3. Include the transition functions in your default style declaration using the transition
    properties, including: transition-property, transition-duration, transition-timing-function,
    and transition-delay. We’ll look at each of these and how they work shortly.
- To support older WebKit browsers, we need to include the -webkit- prefix for all of your transition properties.
- The transition-duration property sets how long the transition will take.
- If we want to change the color at the same time as changing the rotation and size, we can use multiple transitions.
- CSS animations, unlike transitions, allow us to control each step of an animation via keyframes.

Canvas, SVG, and Drag and Drop

- With HTML5’s Canvas API, we are able to draw what we can imagine, all through JavaScript.
- The Canvas API is supported in: Chormoe 4+, Firefox2+, Opera9.6+, Safari3.1+, iOS3.2+, Interner Explorer 9.0+, Android3.0+
- All drawing on the canvas happens via the Canvas JavaScript API
- We can draw shpapes by creating paths. Paths create a blueprint for our line, arcs, and shapes. but paths are invisible until we give them a stroke.
- Saving Canvas Drawing: If we create an image programmatically using the Canvas API, but decide
we’d like to have a local copy of our drawing, we can use the API’s toDataURL method to save our drawing.
toDataURL creates a URL with the image in it, (either a png or jpg, whichever is specified). You can
then right-click the image at this URL, and save it as a PNG or JPEG. - SVG: a specific file format that allows you to describe vector graphics using XML
- We can use SVG to do many of the same tasks we can do with canvas, including drawing
paths, shapes, text, gradients, and patterns. - Raphaël is an open-source JavaScript library that makes drawing and animating with SVG much easier. - Both canvas and SVG allow you to draw custom shapes, paths, and fonts, but

  • Canvas allows for pixel manipulation, as we saw when we turned our video from color to black and white.
  • One downside of canvas is that it operates in what’s known as immediate mode. This means that if you ever
    want to add more to the canvas, you’re unable to simply add to what’s already there.
  • Every time you finish drawing a shape, the canvas no longer has access to that shape,
  • There’s also no access to what’s drawn on the canvas via the DOM
  • Canvas is much faster than SVG
  • Canvas is generally the better choice if you’re looking to design a game requiring lots of animations.
In summary, if you need to paint pixels to the screen and have no concerns about the ability to retrieve and
modify your shapes, canvas is probably the better choice. If, on the other hand, you need to be able to access
and change specific aspects of your graphics, SVG might be more appropriate.
- we need several steps to add drag and drop to our page:
  • Set the draggable attribute on any HTML elements we’d like to be draggable.
  • Add an event listener for the dragstart event on any draggable HTML elements.
  • Add an event listener for the dragover and drop events on any elements that we want to have accept dropped items.
- In order to make the elements draggable we add the draggable attribute to them and set the value to true.
